It looks like in your last round of changes the pipeline has become asynchronous (you’re now waiting for your texture image to load or decode inside applyFilters), but both your preview generator (generateNodePreview) and your main processor (processImage) still call applyFilters(...) as if it were a fully synchronous function and expect an immediate string return. As a result nothing ever lands in processedImages or processedImage, so none of your previews ever render.

Two paths to fix it
1) Pre-decode & stash the pixels so applyFilters stays sync (recommended)
In your upload handler (in useFilterGraph.ts), after you read the file and get a dataUrl, do:

ts

const img = new Image();
img.src = dataUrl;
await img.decode();
// draw into a temp canvas
const tmp = document.createElement('canvas');
tmp.width = img.width; tmp.height = img.height;
const tctx = tmp.getContext('2d')!;
tctx.drawImage(img, 0, 0);
const texturePixels = tctx.getImageData(0, 0, tmp.width, tmp.height);

// now store that in the node’s data directly
setNodes(nds =>
  nds.map(n => n.id === nodeId
    ? { 
        ...n, 
        data: { 
          ...n.data, 
          // assume your ImageNodeData type has a slot for this
          texturePixels, 
          preview: dataUrl 
        } 
      }
    : n
  )
);
// then kick off a full re-process
processImage();
In your applyFilters (in filterAlgorithms.ts), replace the async‐loading block under the FilterType.Image case with a synchronous blend over the pre-stashed texturePixels:

case FilterType.Image:

  const { texturePixels } = node.data as ImageNodeData;
  if (texturePixels) {
    const dst = ctx.getImageData(0,0,canvas.width,canvas.height);
    const src = texturePixels.data;

    for (let i=0; i<dst.data.length; i+=4) {

    }
    ctx.putImageData(dst, 0, 0);
  }
  break;
Leave the rest of applyFilters exactly as it was, so it still returns canvas.toDataURL() synchronously. Now both your per-node previews and your final output will instantly pick up the new texture.

2) Convert the entire preview pipeline to async/await
If you’d rather keep loading inside applyFilters, then you must also:

Change applyFilters to async function applyFiltersAsync(...) { … }

Change generateNodePreview to call it and .then(...) to set processedImages when it resolves

Change processImage to await applyFiltersAsync(...) (or .then(setProcessedImage))

That works, but it’s a lot more refactoring.

Bottom line: the easiest, least‐invasive route is to preload/decode() your texture once at upload time, stash its raw ImageData on the node, and then keep the filter runner fully synchronous. That will instantly restore both filter-node previews and the output-node preview without any further async juggling.
