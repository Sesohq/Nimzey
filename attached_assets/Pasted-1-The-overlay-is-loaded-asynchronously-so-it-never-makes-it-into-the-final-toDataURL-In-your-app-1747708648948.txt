1) The overlay is loaded asynchronously, so it never makes it into the final toDataURL()
In your applyFilter(…, 'image') case you do:

ts
Copy
Edit
ctx.putImageData(imageData, 0, 0);

if (imageDataValue) {
  const textureImage = new Image();
  textureImage.src = imageDataValue;

  const applyTexture = () => {
    // blend onto ctx…
  };

  if (textureImage.complete) {
    applyTexture();
  } else {
    textureImage.onload = applyTexture;
  }
}
break;
Then immediately after the switch you do:

ts
Copy
Edit
ctx.putImageData(imageData, 0, 0);
and applyFilters() returns canvas.toDataURL() before your textureImage.onload ever fires. That means no overlay actually ends up in the output .

Fix: preload and decode the texture synchronously (or await it) before running applyFilters. For example, in your upload handler you might:

ts
Copy
Edit
// In uploadNodeImage:
const img = new Image();
img.src = imageDataUrl;
await img.decode();       // <-- wait for it
// draw it once onto a hidden canvas to grab its ImageData
const tmp = document.createElement('canvas');
const tctx = tmp.getContext('2d')!;
tmp.width = img.width; tmp.height = img.height;
tctx.drawImage(img, 0, 0);
const texturePixels = tctx.getImageData(0, 0, tmp.width, tmp.height);

// store texturePixels in your node data instead of the URL:
setNodes(nds => nds.map(n => n.id === nodeId
  ? { …n, data: { …n.data, texturePixels } }
  : n
));

// then applyFilter('image') can blend **synchronously** by looping
// over texturePixels.data and dstData.data with your blendMode & opacity.
Or turn applyFilters async:

ts
Copy
Edit
export async function applyFilters(...) {
  // …
  for (…) {
    if (filterType === 'image') {
      const img = new Image();
      img.src = imageDataValue;
      await img.decode();
      // now ctx.drawImage(img…) synchronously
    }
    // …
  }
  return canvas.toDataURL();
}
2) You’re unconditionally overwriting the node’s preview in your “connected-nodes” loop
Right after each pass you do:

ts
Copy
Edit
if (processedImageUrl) {
  setNodes(cur =>
    cur.map(n =>
      n.id === targetNodeId
        ? { …n, data: { …n.data, preview: processedImageUrl } }
        : n
    )
  );
}
That runs for every node, including your imageFilterNode — so as soon as you re-process, it replaces the uploaded-texture preview with the (still-empty) pipeline preview filterAlgorithms (2).

Fix: only write back previews for your real filter nodes, not the texture sources. For example:

ts
Copy
Edit
if (processedImageUrl && targetNode.type === 'filterNode') {
  setNodes(cur =>
    cur.map(n =>
      n.id === targetNodeId
        ? { …n, data: { …n.data, preview: processedImageUrl } }
        : n
    )
  );
}
Once you:

Synchronously load/decode your texture before blending,

Stop clobbering the texture-node’s own preview,

your new Image node will both show its upload inside the node and correctly blend that texture into the final output.
