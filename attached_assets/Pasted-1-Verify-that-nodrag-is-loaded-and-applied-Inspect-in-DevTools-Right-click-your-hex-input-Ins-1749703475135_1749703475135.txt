1. Verify that .nodrag is loaded and applied
Inspect in DevTools: Right-click your hex <input> → Inspect → make sure you see class="nodrag …".

Check the CSS: In the console, type

js
Copy
Edit
getComputedStyle(document.querySelector('input.nodrag')).cursor
You should get something like "text", not "move" or "grab".

Ensure stylesheet import (or include in your build):

js
Copy
Edit
import 'reactflow/dist/style.css';
2. Look for a parent wrapper grabbing the event
If your <input> lives inside another custom <div> or HOC that’s listening onMouseDown, it might catch it first. You can:

Give that wrapper className="nodrag" too

Move your <input> out of any extra container

3. Add explicit stopPropagation on the input
If ReactFlow’s handler still fires before .nodrag is checked, you can guarantee the event never bubbles up:

tsx
Copy
Edit
<input
  type="text"
  value={hex}
  onChange={e => onHexChange(e.target.value)}
  onBlur={validateAndFormatHex}
  className="nodrag w-full px-2 py-1 border rounded focus:outline-none"
  onMouseDown={e => e.stopPropagation()}
  onClick={e => e.stopPropagation()}
  draggable={false}      // also prevents native drag concerns
  placeholder="#rrggbb"
/>
draggable={false} is a bonus: it stops any HTML5 drag events trying to hijack your field.

Putting it all together
tsx
Copy
Edit
// CheckerboardNode.tsx
import 'reactflow/dist/style.css';

export function CheckerboardNode({ data }) {
  const [hex, setHex] = useState(data.color);

  return (
    <div className="react-flow__node checkerboard-node">
      {/* …other nodrag-wrapped controls… */}
      <div className="color-input nodrag">
        <input
          type="text"
          value={hex}
          onChange={e => setHex(e.target.value)}
          onBlur={() => validateAndFormatHex(hex, setHex)}
          onMouseDown={e => e.stopPropagation()}
          onClick={e => e.stopPropagation()}
          draggable={false}
          className="nodrag w-full px-2 py-1 border rounded focus:outline-none"
          placeholder="#rrggbb"
        />
      </div>
    </div>
  );
}
With that:

CSS .nodrag tells ReactFlow “skip me.”

stopPropagation ensures no parent sees the click.

draggable={false} prevents any default browser drag behavior.

That combo will guarantee your hex input can be focused and edited, even if ReactFlow’s wrappers are aggressively capturing clicks.
