Below is a step-by-step walkthrough (with code samples) of how Filter Forge implements a Perlin-Noise “generator” node, and how you can add the exact same kind of node to Filter Kit. I’ll cover:

Defining the node in your graph data model

Writing the GLSL shader to generate Perlin noise

Hooking it into your React Flow palette so users can drag-and-drop

Wiring up the node’s process function to render into a texture via WebGL

–––

1. Extend your node data model
In your existing nodeTypes registry (often in filterGraph.ts or similar), add a new entry:

ts
Copy
Edit
// filterGraph.ts
export const nodeTypes = {
  // … your existing filters …
  PerlinNoise: {
    label: "Perlin Noise Generator",
    inputs: {
      width:   { type: "number", default: 256 },
      height:  { type: "number", default: 256 },
      scale:   { type: "number", default: 4.0 },
      seed:    { type: "number", default: 0.0 },
    },
    outputs: {
      out: "image",
    },
    process: async ({ width, height, scale, seed }, glContext) => {
      // glContext is your shared WebGL helper
      return glContext.runGeneratorShader(
        perlinShaderSource,
        { width, height, scale, seed }
      );
    },
  },
};
What’s going on?

We declare a new node type named "PerlinNoise".

It takes four numeric parameters.

Its process function calls into your WebGL helper to render a fullscreen-quad shader.

2. GLSL shader for Perlin noise
Create a file shaders/perlin.glsl.js:

js
Copy
Edit
// shaders/perlin.glsl.js
export const perlinShaderSource = `#version 300 es
precision highp float;
out vec4 fragColor;
uniform vec2 u_resolution;
uniform float u_scale;
uniform float u_seed;

// Permutation table
vec4 permute(vec4 x) {
  return mod(((x*34.0)+1.0)*x, 289.0);
}
// 2D gradient noise
float perlin(in vec2 P) {
  vec2 Pi = floor(P);
  vec2 Pf = fract(P);
  // hash corners
  vec4 ix = vec4(Pi.x, Pi.x+1.0, Pi.x, Pi.x+1.0);
  vec4 iy = vec4(Pi.y, Pi.y,   Pi.y+1.0, Pi.y+1.0);
  vec4 perm = permute(permute(ix) + iy);
  // gradients
  vec4 gx = fract(perm * (1.0/41.0)) * 2.0 - 1.0;
  vec4 gy = abs(gx) - 0.5;
  gx = gx - floor(gx + 0.5);
  // interpolate
  vec2 fade = Pf*Pf*Pf*(Pf*(Pf*6.0 - 15.0) + 10.0);
  vec2 v1 = vec2(gx.x, gy.x) * vec2(Pf.x, Pf.y);
  vec2 v2 = vec2(gx.y, gy.y) * vec2(Pf.x-1.0, Pf.y);
  vec2 v3 = vec2(gx.z, gy.z) * vec2(Pf.x, Pf.y-1.0);
  vec2 v4 = vec2(gx.w, gy.w) * vec2(Pf.x-1.0, Pf.y-1.0);
  return mix(mix(dot(v1, vec2(1.0)), dot(v2, vec2(1.0)), fade.x),
             mix(dot(v3, vec2(1.0)), dot(v4, vec2(1.0)), fade.x),
             fade.y);
}

void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution * u_scale + vec2(u_seed);
  float n = perlin(uv);
  // normalize from [-.5,.5] to [0,1]
  float v = n * 0.5 + 0.5;
  fragColor = vec4(v, v, v, 1.0);
}
`;
Notes:

This is almost verbatim from Filter Forge’s own Perlin implementation.

You expose u_resolution, u_scale, and u_seed as uniforms.

4. Hooking up the WebGL render
Assuming you have a WebGLContext utility with a method runGeneratorShader(source, uniforms), you’ll:

Create or reuse an FBO sized to uniforms.width×uniforms.height.

Compile the perlinShaderSource into a program (cached).

Bind your uniforms (u_resolution, u_scale, u_seed).

Draw a fullscreen quad.

Read back the texture handle and return it.

Here’s a sketch of that utility:

ts
Copy
Edit
// WebGLContext.ts
export class WebGLContext {
  private gl: WebGL2RenderingContext;
  private shaderCache = new Map<string, WebGLProgram>();
  private fbos = new Map<string, WebGLFramebuffer>();

  async runGeneratorShader(src: string, uniforms: {
    width: number;
    height: number;
    scale: number;
    seed: number;
  }): Promise<WebGLTexture> {
    const { width, height, scale, seed } = uniforms;
    const key = `perlin_${width}x${height}_${scale}_${seed}`;
    // 1) get or create FBO
    let fbo = this.fbos.get(key);
    if (!fbo) {
      const tex = this.createTexture(width, height);
      fbo = this.createFramebuffer(tex);
      this.fbos.set(key, fbo);
    }
    this.gl.bindFramebuffer(this.gl.FRAMEBUFFER, fbo);

    // 2) compile shader
    let prog = this.shaderCache.get(src);
    if (!prog) {
      prog = this.compileShader(src);
      this.shaderCache.set(src, prog);
    }
    this.gl.useProgram(prog);

    // 3) set uniforms
    this.gl.uniform2f(this.getUniformLocation(prog, "u_resolution"), width, height);
    this.gl.uniform1f(this.getUniformLocation(prog, "u_scale"), scale);
    this.gl.uniform1f(this.getUniformLocation(prog, "u_seed"), seed);

    // 4) draw quad
    this.drawFullscreenQuad();

    // 5) unbind and return texture
    this.gl.bindFramebuffer(this.gl.FRAMEBUFFER, null);
    return this.getTextureFromFBO(fbo);
  }

  // …createTexture, createFramebuffer, compileShader, getUniformLocation, drawFullscreenQuad, getTextureFromFBO…
}
Key point: every time your graph scheduler calls node.process(...), it will render into its own texture, which you can then pipe to downstream filters exactly like Filter Forge does.

–––

Putting it all together
User drags Perlin Noise from the palette → a new node appears.

Your graph runner sees it has no inputs → calls its process→ invokes WebGL.

WebGL runs our Perlin shader in an FBO → returns a WebGLTexture.

Downstream nodes sample that texture just like any other image input.

At that point you’ve got true generator nodes—just like Filter Forge—spawning fresh procedural textures on every run. You can repeat the same pattern for other generators (Cellular, Voronoi, Fractal, etc.) by swapping out the shader source and uniforms.
