Below is a comprehensive breakdown of how Filter Forge’s Mask (blender) component works under the hood—and how you can reproduce it exactly in Filter Kit.

Summary:
Filter Forge’s Mask node takes two inputs—Source (the image to be masked) and Mask (a black-&-white map or an alpha map)—and multiplies the Source’s RGBA channels by the Mask’s luminance or alpha values. Internally, it queries each component for the pixel color/alpha (via the scripting API), computes the mask value (either the mask’s alpha or its Rec. 709 luminance), and outputs:

ini
Copy
Edit
out.rgb   = src.rgb * mask;
out.alpha = src.alpha * mask;
This behavior mirrors the Mask component in Filter Forge’s C++ code and Blend component behavior when using a mask as a selection.

1. Filter Forge’s Component Model
1.1 Components & Pixel Queries
Every Filter Forge component is a modular “pixel shader” that, when asked, returns its output color at any given UV or pixel coordinate. Internally, components don’t draw pixels directly; instead, the engine “asks” each component for its pixel color or alpha at specific coordinates, then composes results according to the graph topology

1.2 Blend & Result Integration
Masking in Filter Forge often leverages the generic Blend machinery: the engine blends the masked result over transparency or the original image, using the mask as the alpha channel—identical to the Photoshop “Normal” blend behavior with a selection mask

2. Mask Logic in Filter Forge
2.1 Inputs & Parameters
Source: the image being masked

Mask: any gray-scale or alpha-channel input (another map component, Perlin, etc.)

Mode: whether to use the Mask’s alpha channel or its luminance (“Luma Mode”)

These map directly to the component’s Map Inputs in the Filter Editor 
filterforge.com
.

2.2 Computing the Mask Value
Alpha Mode: mask = maskComp.alpha

Luma Mode: mask = rec709_luma(maskComp.rgb)

The Rec. 709 luminance formula is:

ini
Copy
Edit
mask = 0.2126*R + 0.7152*G + 0.0722*B
which is exactly what high-definition video standards (and Filter Forge) use for perceptual brightness

2.3 Final Output per Pixel
Once mask is known, the component returns:

glsl
Copy
Edit
out.rgb   = src.rgb   * mask;
out.alpha = src.alpha * mask;
This hard-multiplication creates a true matte: black areas of the mask fully hide the source, white areas leave it intact

3. Implementing Mask in Filter Kit
3.1 Node Definition
In your filterGraph.ts, add a Mask node type:

ts
Copy
Edit
Mask: {
  label: "Mask",
  inputs: {
    src:   { type: "image"             },
    mask:  { type: "image"             },
    lumaMode: { type: "boolean", default: false },
  },
  outputs: { out: "image" },
  process: async ({ src, mask, lumaMode }, gl) => {
    return gl.runMaskShader(maskShaderSource, {
      srcTexture: src,
      maskTexture: mask,
      useLuma: lumaMode ? 1.0 : 0.0,
    });
  },
},
3.2 GLSL Shader
Create shaders/mask.glsl.js:

js
Copy
Edit
export const maskShaderSource = `#version 300 es
precision highp float;
uniform sampler2D u_src, u_mask;
uniform vec2      u_resolution;
uniform float     u_useLuma;
out vec4 fragColor;

// Rec.709 luminance
float getLuma(vec3 c) {
  return dot(c, vec3(0.2126, 0.7152, 0.0722));
}

void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 s = texture(u_src,  uv);
  vec4 m = texture(u_mask, uv);
  float maskVal = mix(m.a, getLuma(m.rgb), u_useLuma);
  fragColor = vec4(s.rgb * maskVal, s.a * maskVal);
}`;
Notes:

We expose useLuma as a 0/1 flag to switch between alpha and luma modes.

Luma is computed with exact Rec. 709 weights for perfect fidelity

4. Integrating into WebGL Runner
Reuse your existing runGeneratorShader pattern—rename it runMaskShader—to:

Bind both source and mask textures to texture units.

Set uniforms u_resolution, u_useLuma.

Draw a full-screen quad.

Return the resulting FBO’s texture for downstream sampling.

This matches Filter Forge’s internal C++ pipeline, where mask and source are sampled and multiplied in one pass

.
