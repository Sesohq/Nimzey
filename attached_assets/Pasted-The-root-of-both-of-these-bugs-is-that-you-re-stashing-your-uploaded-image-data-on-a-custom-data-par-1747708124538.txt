The root of both of these bugs is that you’re stashing your uploaded image data on a custom data.parameters field, but the core filter-graph runner (and applyFilters) only ever reads from the built-in node.data.params array. So:

Overlay never actually makes it into the chain, because when it walks that chain it only sees your original source node’s image (and ignores your new upload).

On any “refresh” or re-preview you throw away your custom data.parameters and re-hydrate from the original data.params, so your node’s preview falls back to whatever the source node shows.

1. Update your upload handler to write into node.data.params
In your uploadNodeImage callback (inside useFilterGraph.ts), replace your parameters–based update with something like this:

ts
Copy
Edit
const uploadNodeImage = useCallback((nodeId: string, file: File) => {
  const reader = new FileReader();
  reader.onload = (e) => {
    const imageDataUrl = e.target?.result as string;

    setNodes(prev =>
      prev.map(node => {
        if (node.id !== nodeId) return node;

        // update the *params* array, not a custom `parameters` object
        const newParams = (node.data as ImageNodeData).params.map(p => {
          if (p.name === 'imageData') {
            return { ...p, value: imageDataUrl };
          }
          return p;
        });

        return {
          ...node,
          data: {
            ...node.data,
            params: newParams,
            // also update the “preview” so the node UI shows it immediately
            preview: imageDataUrl
          }
        };
      })
    );

    // re‐run the graph so the new texture actually bubbles through to the output
    processImage();
  };
  reader.readAsDataURL(file);
}, [setNodes, processImage]);

2. Keep your Image nodes out of “reset” or re-preview loops
You’re probably seeing your upload get blown away because on canvas refresh you’re doing something like:

ts
Copy
Edit
// in resetCanvas…
setNodes(initialNodesRef.current);
That blows away every upload. You have two easy fixes:

Don’t call a full resetCanvas() on mere background-clicks — reserve it for “start over” only.

If you must re-hydrate, merge in existing params/previews for your image nodes:

ts
Copy
Edit
const resetCanvas = useCallback(() => {
  setNodes(prev => 
    initialNodesRef.current.map(init => {
      const existing = prev.find(n => n.id === init.id);
      if (init.type === 'imageFilterNode' && existing) {
        return {
          ...init,
          data: {
            ...init.data,
            // carry forward the live upload & preview
            params: (existing.data as ImageNodeData).params,
            preview: (existing.data as ImageNodeData).preview
          }
        };
      }
      return init;
    })
  );
  processImage();
}, [processImage]);
TL;DR
Write your uploaded URL into node.data.params (that’s what the filter runner reads), not a custom data.parameters field.

Re-run the graph after your upload so your texture makes it into the output.

Don’t blow away your node’s params and preview on every canvas reset — either stop calling resetCanvas() so often, or merge existing uploads back in when you do.

With those two changes in place your extra image-node will:

Immediately blend into the output.

Keep showing your upload any time you rerun or refresh the canvas.
