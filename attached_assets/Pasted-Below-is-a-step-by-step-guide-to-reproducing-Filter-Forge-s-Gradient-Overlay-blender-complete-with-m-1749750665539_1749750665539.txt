Below is a step-by-step guide to reproducing Filter Forge’s Gradient Overlay blender—complete with mask interaction—inside Filter Kit.

Summary
Filter Forge implements gradient overlays by first generating a procedural gradient (using its Free Gradient or Profile Gradient components), then blending that gradient over the source image with a chosen blend mode (e.g. “Overlay”), and finally allowing an optional mask to modulate the blend amount per pixel. We’ll replicate this by:

Defining a Gradient Generator node (mirroring Free Gradient)

Writing its GLSL shader (linear interpolation between two colors along a direction)

Defining a Gradient Overlay Blend node that takes the generated gradient and source image, applies the Overlay blend formula, and respects an optional mask input

Implementing the blend shader (per-channel Overlay math)

Wiring everything through your React Flow + WebGL runner exactly as in Filter Forge

2. GLSL Shader for Gradient
js
Copy
Edit
// shaders/gradient.glsl.js
export const gradientShaderSource = `#version 300 es
precision highp float;
out vec4 fragColor;
uniform vec2  u_resolution;
uniform vec4  u_color1, u_color2;
uniform float u_angle, u_type;

void main() {
  vec2 uv = (gl_FragCoord.xy / u_resolution) * 2.0 - 1.0;
  float t;
  if (u_type < 0.5) {
    // Linear
    vec2 dir = vec2(cos(u_angle), sin(u_angle));
    t = dot(uv, dir) * 0.5 + 0.5;
  } else if (u_type < 1.5) {
    // Radial
    t = length(uv);
  } else {
    // Angular
    t = atan(uv.y, uv.x) / 3.1415926 * 0.5 + 0.5;
  }
  t = clamp(t, 0.0, 1.0);
  fragColor = mix(u_color1, u_color2, t);
}`;
This matches Filter Forge’s interpolation logic for linear, radial, and angular gradients

4. GLSL Shader for Overlay + Mask
js
Copy
Edit
// shaders/blend.glsl.js
export const blendShaderSource = `#version 300 es
precision highp float;
uniform sampler2D u_src, u_blend, u_mask;
uniform vec2      u_resolution;
uniform float     u_mode;
out vec4 fragColor;

vec3 blendOverlay(vec3 A, vec3 B) {
  return mix(
    2.0*A*B,
    1.0 - 2.0*(1.0-A)*(1.0-B),
    step(0.5, A)
  );
}

void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 S = texture(u_src,   uv);
  vec4 B = texture(u_blend, uv);
  float M = (u_mask>0.5) 
            ? texture(u_mask, uv).r 
            : 1.0;

  vec3 C;
  if (u_mode < 0.5) {
    // Normal
    C = mix(S.rgb, B.rgb, B.a);
  } else {
    // Overlay
    C = blendOverlay(S.rgb, B.rgb);
  }
  fragColor = vec4(mix(S.rgb, C, M), S.a);
}`;

5. React Flow + WebGL Integration
Palette: “Gradient Generator” and “Gradient Overlay” auto-appear from your nodeTypes.

Drag-&-Drop: Already implemented per earlier patterns.

WebGL Runner:

runGeneratorShader for gradients

runBlendShader for overlay blends

Both allocate FBOs, compile/cached shaders, bind uniforms, draw fullscreen quads, and return textures

Mask Input: Optional—if absent, M=1.0 yields uniform overlay; if present, it modulates opacity per pixel exactly like Filter Forge

6. Testing & Edge Cases
No Mask: gradient overlays fully according to blend mode

Hard Mask: black pixels → 0 effect, white → 100% overlay

Semi-transparent Mask: partial overlay

Different Modes: switch u_mode to test “Normal” vs. “Overlay” vs. “Multiply”

Gradient Types: linear, radial, angular—verify angle and tiling
