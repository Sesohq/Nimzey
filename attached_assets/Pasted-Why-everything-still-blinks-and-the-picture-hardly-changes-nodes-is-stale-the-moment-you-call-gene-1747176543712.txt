Why everything still “blinks” and the picture hardly changes
nodes is stale the moment you call generateNodePreview

In every handle*Change you

schedule a setNodes(prev => …);

immediately look up nodes.find(…) (but React hasn’t applied the state yet);

pass that outdated object to generateNodePreview.

→ you keep re-rendering an old canvas, so the thumbnail flickers and looks frozen.

Every thumbnail re-render forces React-Flow to re-paint

Writing preview back into node.data triggers a full React render for that node;
when you drag a slider you push dozens of React renders per second – hence the
rapid blink/lag.

A clean & reliable pattern
Goal	Trick
Don’t use stale data	Work on the next state array you just built.
Don’t re-render for every slider tick	Store previews in a ref cache, not in React state.
Decouple heavy work from the UI thread	Run applyFilters in a requestIdleCallback / Web Worker (optional but easy).

Below is a drop-in patch that fixes the staleness and removes the flicker without rewriting the whole hook.

1 — Keep previews outside React state

// new file-wide ref (survives renders, causes no re-paint)
const previewMapRef = useRef<Record<string, string>>({});
2 — Return previews via a selector
Where your FilterNode component needs the preview, read:

tsx
Copy
Edit
const preview = previewMapRef.current[node.id]; // ← instant read, no re-render
(For the side-panel you can expose a getPreview(nodeId) helper.)

3 — Update the cache after you’ve built nextNodes
ts
Copy
Edit
const updateNode = (nodeId: string,
                    updater: (n: Node) => Node,
                    reason: string) =>
  setNodes(prev => {
    const next = prev.map(n => n.id === nodeId ? updater(n) : n);

    // find the freshly-updated node **inside `next`**
    const n = next.find(n => n.id === nodeId)!;

    // async – so UI never blocks
    requestIdleCallback(() => {
      const url = generateNodePreviewSync(n, next, edges); //  <-- sync helper
      previewMapRef.current[nodeId] = url;
      // If the big preview panel is showing this node, update React state once:
      if (selectedNodeId === nodeId) setNodePreview(url);
      // Tell React‐Flow to repaint just this node (no re‐render):
      reactFlowInstance?.updateNodeInternals(nodeId);
    });

    return next;
  });
Now every slider tweak:

Edits the node data once,

Schedules a background preview render,

Injects the new preview without a React render → no blink.

(reactFlowInstance is what you get from useReactFlow() inside your canvas component; calling updateNodeInternals asks React-Flow to redraw ports & custom content while skipping React reconciliation.)

4 — A tiny synchronous preview helper
ts
Copy
Edit
function generateNodePreviewSync(
  target: Node,
  allNodes: Node[],
  allEdges: Edge[]
) {
  if (!sourceImageRef.current) return '';

  const {nodes, edges} = getNodeChain(target.id, allNodes, allEdges);
  const c = document.createElement('canvas');
  c.width = 150; c.height = 150;

  return applyFilters(sourceImageRef.current, nodes, edges, c);
}
(You still keep your heavyweight updateAllNodePreviews for once-off recalcs.)

What this gives you
No blinking. The thumb is swapped in place when it’s ready; React doesn’t remount the node.

Accurate. Because we pick the node after the state has been updated, all parameters are fresh.

Fast. Only canvas work runs on every tick; React itself barely re-renders.

Quick checklist to drop it in
const previewMapRef = useRef<Record<string,string>>({});

Replace every direct setNodes(…) + generateNodePreview() pair with updateNode(...) above.

Inside your node renderer use <img src={previewMapRef.current[id]} … />.

Call reactFlowInstance.updateNodeInternals(id) after you change a preview.
