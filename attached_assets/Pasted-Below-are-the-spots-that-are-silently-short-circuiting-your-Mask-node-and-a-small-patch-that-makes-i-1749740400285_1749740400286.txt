Below are the spots that are silently short-circuiting your Mask node and a small patch that makes it behave exactly like Filter Forge / Photoshop layer-masks.

1 Nothing is reaching the mask stage
applyFilter() only runs the masking math when both of these are true:

The node is of type mask

nodeData.maskImageData exists (see the guard) filterAlgorithms (2)

If the Perlin Noise (or any generator) is not wired into the “Mask” handle of the Mask node, maskImageData is never injected and the branch is skipped, so you always get an untouched source.
Double-check in the editor that you have a solid connection line from Perlin Noise → Mask (mask handle); the dotted outline you showed in the screenshot means no connection, so the filter is being bypassed.

2 maskImageData never survives the spread
Inside the loop that builds nodeDataWithMask you currently have:

ts
Copy
Edit
nodeDataWithMask = {
  .node.data,
  maskImageData: maskImageData.data
};
The leading dot is a typo – the object spread never happens, so the whole object becomes undefined the moment TypeScript strips the illegal token. Replace it with the proper spread operator:

ts
Copy
Edit
nodeDataWithMask = {
  ...node.data,
  maskImageData: maskImageData.data
};
``` :contentReference[oaicite:1]{index=1}

Without this fix `applyFilter()` still fails the guard in step 1.

## 3  Alpha is applied, but you never see it  
`applyMaskFilterCanvas()` multiplies only the alpha channel:

```ts
data[i + 3] = Math.round(sourceA * maskValue);  // alpha
``` :contentReference[oaicite:2]{index=2}

That’s correct, but the preview you render later shows the canvas on a black backdrop, so fully transparent pixels look black – not the “hole” you expect.

### Quick preview tweak  
After the mask node finishes, draw a checkerboard or another contrasting background behind the layer when `ctx.globalAlpha < 1`. That visual cue will confirm the mask is working.

## 4  Safer (and faster) masking helper  
If you prefer to keep everything CPU-side for now, drop-in this trimmed worker that:

* Resizes the mask once (with `drawImage`) instead of per-pixel math  
* Handles both **“Black hides / White shows”** and **Luma invert** modes  
* Works whether the mask is opaque or already has its own alpha

```ts
function applyMaskFilterCanvas(
  src: ImageData,
  mask: ImageData,
  invert = false,           // lumaMode checkbox
) {
  const s = src.data, m = mask.data;
  for (let i = 0; i < s.length; i += 4) {
    // Per-pixel luma from the mask
    const lum = (0.3 * m[i] + 0.59 * m[i + 1] + 0.11 * m[i + 2]) / 255;
    const a   = invert ? 1 - lum : lum;

    // RBGA
    s[i + 3] = s[i + 3] * a;       // premultiply alpha
  }
}
Drop-in call:

ts

const srcData  = ctx.getImageData(0,0,w,h);
const maskData = maskCtx.getImageData(0,0,w,h);

applyMaskFilterCanvas(srcData, maskData, useLuma);
ctx.putImageData(srcData, 0, 0);
It eliminates all the nested loops you currently run in applyMaskFilterCanvas() and guarantees the alpha math matches Photoshop 1-for-1.
