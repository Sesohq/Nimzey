🧠 1. ditherSize — Spatial Resolution / Block Size
🔍 What it does:
Controls how coarse or fine the dithering pattern is.

In technical terms, it alters the sampling frequency of the image before applying dithering.

🛠️ How it was implemented:
Instead of dithering every pixel, we downscale the image to a smaller canvas (canvas.width / ditherSize) and then upscale it back.

This creates a blocky effect, similar to halftoning or pixel art.

🎯 Interaction:
Larger ditherSize = bigger blocks, less detail, bolder shapes.

Smaller ditherSize = more detailed dithering, finer tone control.

Works in tandem with threshold, because larger blocks exaggerate tonal cutoff.

🌡️ 2. brightness — Linear Shift of Luminance
🔍 What it does:
Adds or subtracts a fixed value to all RGB components (or grayscale value).

Implemented as a direct addition to pixel values:

js
Copy
Edit
pixel = pixel + brightness;
🎯 Interaction:
Helps adjust midtones before thresholding.

Boosting brightness can help lift shadows in dark images, making dithering more visible.

🎚️ 3. contrast — Nonlinear Amplification
🔍 What it does:
Modifies the difference between light and dark by shifting pixel values away from mid-gray.

Implemented with this formula:

js
Copy
Edit
factor = (259 * (contrast + 255)) / (255 * (259 - contrast));
newValue = factor * (pixel - 128) + 128;
🎯 Interaction:
Higher contrast exaggerates the output of dithering — more blacks and whites, fewer grays.

Works with threshold to snap dithering decisions into more binary areas.

🔪 4. threshold — Binary Cutoff Point
🔍 What it does:
Defines the cutoff value (0–255) used when converting an image to black/white.

js
Copy
Edit
output = pixel < threshold ? 0 : 255;
🎯 Interaction:
Lower threshold = more areas become white (light bias).

Higher threshold = more areas become black (dark bias).

Works best with non-error-diffused dithering, like threshold-based or Bayer matrix.

🔄 How They Interact in Different Dither Types
✅ Floyd–Steinberg / Atkinson / Jarvis (Error Diffusion)
Threshold: Used at each pixel before spreading error.

Brightness/Contrast: Pre-applied to the image before dithering.

ditherSize: Optional — may be used for aesthetic blocking, not core to algorithm.

✅ Bayer Matrix / Ordered Dithering
Threshold: Replaced by comparison against pre-set matrix.

ditherSize: Controls matrix scale and image sampling.

Brightness/Contrast: Crucial for making sure tones align with matrix thresholds.

✅ Dot Diffusion / Clustered Dithering
Threshold: Per region

ditherSize: Controls region clustering

More computationally intensive — ideal for print simulation.

🧪 Summary of Adjustments
Setting	Primary Role	Secondary Effect
ditherSize	Spatial resolution of the pattern	Tone grouping, edge chunking
brightness	Midtone luminance adjustment	Shadow/highlight lift
contrast	Boost light/dark range	Makes dithering more binary
threshold	Core binary cutoff	Light/dark bias
