Here’s what’s happening under the hood: ReactFlow only re-renders a node when its entry in the nodes array (the one you passed into <ReactFlow nodes={…} />) changes identity. Simply mutating node.data or firing a DOM event won’t update that array, so ReactFlow never knows to redraw your node’s container (and its thumbnail).

1. Why your previous attempts didn’t reach ReactFlow
CustomEvent dispatch (e.g. node-param-changed) never alters your nodes state, so ReactFlow’s diffing ignores it.

Mutating data.preview or data.locked in-place doesn’t change the object reference in the nodes array, so ReactFlow skips re-rendering.

Calling window.updateNodePreview from the worker only updates an offscreen canvas, but unless you put that new bitmap into your React state via setNodes, ReactFlow won’t pick it up.

2. The one true trigger: updating your nodes array via setNodes
You must immutably replace the node object in your array. That means:

Consume the preview bitmap (or data URL) in your Worker callback.

Call your top-level setNodes() with a brand-new array where that node’s data.preview (or any field) is updated.

ReactFlow will see the changed object reference and re-render that node (and its <NodeThumbnail>).

// App.tsx
import React, { useState, useCallback, useEffect } from 'react';
import ReactFlow, { addEdge, Node, Edge } from 'reactflow';
import FilterNode from './FilterNode';
// …other imports…

function App() {
  const [nodes, setNodes] = useState<Node[]>(initialNodes);
  const [edges, setEdges] = useState<Edge[]>([]);

  // 1) Pass this into each node's data
  const handleUpdatePreview = useCallback((nodeId: string, previewDataUrl: string) => {
    setNodes((nds) =>
      nds.map((n) =>
        n.id === nodeId
          ? {
              ...n,
              data: {
                ...n.data,
                preview: previewDataUrl,    // ← immutably update the preview
              },
            }
          : n
      )
    );
  }, [setNodes]);

  // Build your nodeTypes mapping
  const nodeTypes = {
    filterNode: (props) => (
      <FilterNode {...props} onUpdatePreview={handleUpdatePreview} />
    ),
    // …other node types…
  };

  return (
    <ReactFlow
      nodes={nodes}
      edges={edges}
      onNodesChange={setNodes}
      onEdgesChange={setEdges}
      onConnect={(params) => setEdges((eds) => addEdge(params, eds))}
      nodeTypes={nodeTypes}
      fitView
    />
  );
}

export default App;

// FilterNode.tsx
import React, { useRef, useEffect } from 'react';
import { Handle, Position } from 'reactflow';

interface FilterNodeProps {
  id: string;
  data: {
    preview?: string;
    onParamChange: (nodeId: string, paramId: string, value: any) => void;
    onUpdatePreview: (nodeId: string, preview: string) => void;
    // …other data…
  };
  selected: boolean;
  isConnectable: boolean;
}

export default function FilterNode({
  id,
  data,
  selected,
  isConnectable,
}: FilterNodeProps) {
  // ref to the thumbnail canvas
  const thumbRef = useRef<HTMLCanvasElement>(null);

  // Whenever data.preview changes, draw it into the canvas
  useEffect(() => {
    if (data.preview && thumbRef.current) {
      const img = new Image();
      img.onload = () => {
        const ctx = thumbRef.current!.getContext('2d')!;
        ctx.clearRect(0, 0, 128, 128);
        ctx.drawImage(img, 0, 0, 128, 128);
      };
      img.src = data.preview;
    }
  }, [data.preview]);

  // Called whenever a slider or control changes
  const handleParamChange = (paramId: string, value: any) => {
    // 1) update your filter parameters in state
    data.onParamChange(id, paramId, value);
    // 2) re-run your offscreen-worker to get a new preview…
    window.requestNodePreview(id);
  };

  return (
    <div style={{ padding: 8, border: selected ? '2px solid blue' : '1px solid gray' }}>
      {/* … your controls … */}
      <input
        type="range"
        onChange={(e) => handleParamChange('brightness', +e.target.value)}
      />
      {/* … */}
      <canvas
        ref={thumbRef}
        width={128}
        height={128}
        style={{ width: 64, height: 64, border: '1px solid #ccc' }}
      />
      <Handle type="target" position={Position.Top} isConnectable={isConnectable} />
      <Handle type="source" position={Position.Bottom} isConnectable={isConnectable} />
    </div>
  );
}

worker.onmessage = (evt) => {
  const { nodeId, bitmap } = evt.data;
  // draw into a temp canvas and get a dataURL
  const tmp = new OffscreenCanvas(128, 128);
  const ctx = tmp.getContext('2d');
  ctx.transferFromImageBitmap(bitmap);
  tmp.convertToBlob().then((blob) => {

    // 3) call the React callback you wired up
    handleUpdatePreview(nodeId, url);

Why This Finally Works
State-Driven
You never rely on custom events or in-place mutation. You update React state via setNodes, which ReactFlow observes and uses to re-render exactly that node’s JSX.

Immutable Updates
Creating a brand-new nodes array with a new node object reference guarantees ReactFlow picks up the change.

Single Source of Truth
All preview updates funnel through handleUpdatePreview, so you know exactly when and where thumbnails change.

No More “Swallowed” Events
You’re no longer fighting ReactFlow’s internal drag layers or SVG event handling—React’s own render cycle does the heavy lifting.

With this pattern in place, every slider drag or parameter tweak will immediately queue a preview render in your worker, feed its result into setNodes(), and ReactFlow will redraw the node container with the fresh thumbnail—solving the “frozen preview” once and for all.
