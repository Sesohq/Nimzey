You want node previews to update immediately when you adjust sliders, without having to click on the background to force a refresh. Let me propose a solution that should work given what I see in your codebase.
The core issue appears to be that you have two parallel preview systems:

The WebGLPreviewPanel/Canvas that updates in real-time
The per-node thumbnails that only update on pane clicks

Let's implement a pub/sub event system to ensure node thumbnails update immediately during interactions:
typescript// Create a new file: src/lib/previewBus.ts
type PreviewHandler = (nodeId: string, url: string) => void;
const handlers = new Set<PreviewHandler>();

export function onPreview(fn: PreviewHandler) { handlers.add(fn); }
export function offPreview(fn: PreviewHandler) { handlers.delete(fn); }
export function emitPreview(nodeId: string, url: string) {
  handlers.forEach(fn => fn(nodeId, url));
}
Now, let's modify your FilterNode component to subscribe to these events:
typescript// Modify FilterNode.tsx
import { memo, useState, useRef, useMemo, useEffect } from 'react';
import { onPreview, offPreview } from '../lib/previewBus';
// Rest of your imports...

const FilterNode = ({ data, selected, id }: NodeProps<FilterNodeData>) => {
  // Add local state for the preview thumbnail - fallback to data.preview
  const [previewThumb, setPreviewThumb] = useState(data.preview || '');
  
  // Also update local state if data.preview changes (for initial load)
  useEffect(() => {
    if (data.preview) {
      setPreviewThumb(data.preview);
    }
  }, [data.preview]);
  
  // Subscribe to preview updates for this node
  useEffect(() => {
    const handlePreviewUpdate = (nodeId: string, url: string) => {
      if (nodeId === id) {
        setPreviewThumb(url);
      }
    };
    
    // Register the handler
    onPreview(handlePreviewUpdate);
    
    // Clean up on unmount
    return () => {
      offPreview(handlePreviewUpdate);
    };
  }, [id]);
  
  // Rest of your code...
  
  // Use previewThumb instead of data.preview in your render
  {previewThumb && (
    <div className="mb-3">
      <div className="relative border border-gray-200 rounded overflow-hidden" style={{ height: '100px' }}>
        <img 
          src={previewThumb} 
          alt={`${data.label} preview`}
          className="w-full h-full object-cover"
        />
      </div>
    </div>
  )}
  
  // Rest of your component...
}
Finally, modify your WebGLFilterCanvas to emit preview events when it generates a thumbnail:
typescript// Modify WebGLFilterCanvas.tsx
// Near your onPreviewGenerated callback:

// If a selected node is provided, generate node preview
if (selectedNodeId) {
  const nodePreview = await rendererRef.current!.getNodePreview(selectedNodeId, 300);
  if (nodePreview) {
    // Emit the preview update through the bus
    import { emitPreview } from '@/lib/previewBus';
    emitPreview(selectedNodeId, nodePreview);
    
    // Also call the callback if provided
    if (onPreviewGenerated) {
      onPreviewGenerated(nodePreview);
    }
  }
}
This approach has several advantages:

It decouples the thumbnail updates from React Flow's rendering lifecycle
Each node manages its own thumbnail state independently
Updates flow directly through an event system, bypassing potential React Flow caching issues
It preserves your existing preview generation code, just adds a new notification system

This should allow your node thumbnails to update immediately when sliders change, without requiring background clicks to refresh the view.
