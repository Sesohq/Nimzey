2 Why targetHandle can be empty (and how to fix it)
React Flow only attaches sourceHandle / targetHandle to an edge when the
handle elements have explicit id props on both sides.

tsx
Copy
Edit
// Mask node definition
<Handle type="target" position={Position.Left} id="mask" />
<Handle type="target" position={Position.Left} id="source" y={20} />
If either id is missing, React Flow still draws the connection but serialises
the edge as:

ts
Copy
Edit
{
  id: 'reactflow__edge-perlin-output-mask',
  source: 'perlin',
  target: 'mask',
  sourceHandle: 'output',     // OK
  targetHandle: null          //  <-- NULL!
}
Your new filter logic (e.targetHandle === 'mask') therefore never matches and
you fall back to “No mask input connected”.

Action

Open every node component that can accept multiple inputs and verify the
<Handle> tags all have a unique string id.

Restart; the debug block in §1 should now print both connections.

3 Guard against missing handle IDs (future-proof)
Even with proper IDs it’s wise to fall back to positional matching so your node
doesn’t collapse when someone accidentally drags an edge into the wrong port.

Replace your single-line selector with:

ts
Copy
Edit
const maskEdge = edges.find(e =>
  e.target === node.id &&
  (
    e.targetHandle === 'mask'            // preferred
    || (!e.targetHandle && e.source !== sourceImageNodeId)
  )
);
That keeps today’s deterministic behaviour but still works if
targetHandle somehow comes in as null.

4 Re-enable output drawing if everything is connected
Finally, make sure nothing short-circuits after the mask pixel loop.

ts
Copy
Edit
applyMaskFilterCanvas(imgData, maskData, useLuma);
/*  <-- do NOT return here */
ctx.putImageData(imgData, 0, 0);           // always commit
If you call return (or forget putImageData) right after the helper the
canvas stays blank and you’ll again see “no result”.
